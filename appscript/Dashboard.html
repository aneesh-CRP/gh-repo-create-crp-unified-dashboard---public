Connect to Google Sheets for Daily Auto-Refresh

Your CTMS generates reports that land in a Google Drive folder as Google Sheets. Follow this guide to connect them to this dashboard so it refreshes automatically every morning.

Step 1

Open Your Report in Google Sheets

Go to your Looker Reports folder in Google Drive and open the latest "Upcoming Scheduled Appointments" sheet.

Step 2

Publish to the Web as CSV

Go to File → Share → Publish to web. Select the correct sheet tab. Choose CSV format. Click Publish. Copy the URL.

File → Share → Publish to web
→ Select sheet tab
→ Format: .csv
→ Publish → Copy URL
Step 3

Repeat for No-Show Report

Do the same for the "No Show and Canceled Visits" sheet. You'll get two URLs total.

Step 4

Set Up Daily Auto-Refresh

In your Sheet: Extensions → Apps Script. Add a time-driven trigger to run daily at 6am. This forces a re-pull from your CTMS every morning.

Extensions → Apps Script
→ Triggers (clock icon)
→ Add Trigger
→ Time-driven · Day timer · 6am
How Auto-Refresh Works
Daily at 6am: CTMS → Google Sheets
Your Apps Script trigger re-pulls the latest data from your CTMS into the Google Sheet. New appointments and cancellations from the previous day will appear.
When You Open This Dashboard: Sheets → Dashboard
Every time you open this HTML file in your browser, it fetches the latest CSV from Google Sheets automatically. No manual steps needed.
Zero Maintenance Once Connected
After the initial setup (~10 minutes), the entire pipeline is automatic. Open the dashboard each morning and you'll see yesterday's data already loaded.
Troubleshooting
"Failed to load data" error
Make sure you used File → Share → Publish to web (not just "Share link"). The sheet must be publicly published for the CSV URL to work.
Data looks outdated
Check that the Google Sheet is still being updated. The BQ Cloud Function refreshes every time it's called, but the Sheets CSV feed depends on the Apps Script sync running on schedule.
