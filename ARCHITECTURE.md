# CRP Unified Intelligence Dashboard — Architecture & Session Notes

> Last updated: March 12, 2026
> For future Claude sessions or developers picking this up

## Overview

Single-page dashboard (`index.html`, ~970KB) serving CRP clinical research operations data. Deployed on two platforms in parallel:

- **GitHub Pages** (primary): https://aneesh-crp.github.io/gh-repo-create-crp-unified-dashboard---public/
- **Google Apps Script** (SSO-protected): deployed as a web app restricted to `phillyresearch.com` domain

## Apps Script Architecture

The dashboard is too large (~970KB) for Apps Script's HtmlService, so we use an **async chunked loading** pattern:

1. `doGet()` serves `Dashboard.html` (~263KB, HTML/CSS shell only) via `createHtmlOutputFromFile`
2. Client-side loader calls `google.script.run.getDashboardJSChunk(0..3)` in parallel
3. Each chunk is <180KB (under google.script.run's ~256KB return limit)
4. Once all 4 chunks arrive, they're concatenated and injected as a `<script>` element
5. Data fetching routes through `google.script.run.proxyFetch(url)` → server-side `UrlFetchApp` to bypass iframe CSP

## Deploy Pipeline

**GitHub Pages:** Automatic via `.github/workflows/deploy.yml` on push to main.

**Apps Script:** Automatic via `.github/workflows/deploy-appscript.yml`:
1. Triggered on push to main (paths: index.html, appscript/**, scripts/**)
2. Runs `python3 scripts/build-appscript.py` to generate Dashboard.html + JS chunks
3. `clasp push --force` uploads to Apps Script project
4. `clasp deploy` creates a new versioned deployment

**Secret required:** `CLASPRC_JSON` in GitHub repo settings (OAuth credentials from `~/.clasprc.json`)

## Data Sources

All data is fetched from Google Sheets published as CSV. On GitHub Pages, standard `fetch()` is used. On Apps Script, fetches route through `google.script.run.proxyFetch()` using the `IS_APPS_SCRIPT` detection flag.

## localStorage Keys

- `crp_coord_snapshot_v1` — Coordinator visit data snapshot
- `crp_dismissed_actions_v1` — Dismissed action items (60-day auto-prune)
- `crp_coll_[invoice]` — Invoice collection status/notes
- `crp_clickup_token` — ClickUp API token (user-entered)
- `crp_fin_auth` (sessionStorage) — Finance PIN auth flag

## Coordinator Visit History

Three-layer merge system (highest count wins):
1. **coord-history.json** (permanent baseline, committed to repo)
2. **localStorage snapshot** (recent browser-cached data)
3. **Live CRIO data** (fetched at runtime)

Export button (⬇) in Coordinator Visit Snapshot header downloads merged history as JSON.

## New Machine Setup

1. Clone repo
2. `npm install -g @google/clasp`
3. `clasp login` (opens browser for Google OAuth)
4. Copy `.clasp.json` to `appscript/` with the Script ID above
5. Everything else auto-deploys via GitHub Actions
